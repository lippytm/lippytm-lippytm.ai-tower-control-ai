# Repeatability Scorecard Template

> Use this scorecard to evaluate whether a sandbox pattern, simulation packet, multimedia flow, reasoning method, or diagnostic method is strong enough to be reused or promoted.

---

## 1. Score Areas
| Score Area | Description | Suggested Scale |
|---|---|---|
| Input Clarity | How clearly the inputs and boundaries are defined | 1-5 |
| Run Consistency | How consistently the sandbox can be rerun | 1-5 |
| Observation Quality | How useful the observed outputs are | 1-5 |
| Reuse Strength | How well the pattern adapts to other repos or domains | 1-5 |
| Documentation Quality | How clearly assumptions, limits, and notes are documented | 1-5 |
| Promotion Readiness | How ready the pattern is for packaging or broader use | 1-5 |
