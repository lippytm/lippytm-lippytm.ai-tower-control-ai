# Repo to Monetization Pathway Guide

## Purpose
This guide defines how to map repositories to clearer monetization pathways across the lippytm ecosystem.

## Core Questions
For each repository ask:
- what valuable asset does this repo contain?
- what product, service, or application could be built from it?
- what educational or media surfaces could strengthen it?
- what CRM, reporting, or platform support is needed?
- what related repos can reuse the same pathway?
- what review level applies to the delivery surfaces?

## Suggested Mapping Fields
- repo ID
- repo category
- linked product IDs
- linked service IDs
- linked application IDs
- linked platform IDs
- linked concept IDs
- linked component IDs
- monetization category
- maturity stage
- priority wave
- notes

## Suggested Monetization Categories
- product monetization
- service monetization
- application monetization
- educational-business monetization
- media and campaign monetization
- CRM growth monetization
- cross-domain portfolio monetization

## Best Practices
- map highest-value repos first
- allow one repo to support multiple monetization categories when appropriate
- connect each pathway back to stable IDs and trackers
- distinguish internal enablement value from external delivery value
- review monetization mappings after major product or architecture changes
