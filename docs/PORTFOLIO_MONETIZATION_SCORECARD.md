# Portfolio Monetization Scorecard

## Purpose
This scorecard helps compare monetization pathways, packages, and bundles across the lippytm ecosystem.

## Score Areas
| Score Area | Description | Suggested Scale |
|---|---|---|
| Asset Strength | Quality and usefulness of the underlying repos, products, services, or applications | 1-5 |
| Packaging Clarity | How clearly the monetization path is defined | 1-5 |
| Cross-Domain Reuse | How well the path supports more than one domain | 1-5 |
| Platform Fit | How well delivery surfaces match the package | 1-5 |
| Documentation and Governance | How clearly review, documentation, and transparency are handled | 1-5 |
| Portfolio Leverage | How much the package strengthens the broader portfolio | 1-5 |
| Expansion Potential | How easily it can be improved, bundled, or reused later | 1-5 |

## Interpretation Guidance
- **29-35**: strong monetization candidate
- **22-28**: good candidate with targeted improvement needed
- **15-21**: partial package, more shaping needed
- **below 15**: early-stage path that should remain in development

## Best Practices
- score both single-asset paths and bundled paths
- use the scorecard to choose what gets wave priority
- compare scores across products, services, applications, and bundles
