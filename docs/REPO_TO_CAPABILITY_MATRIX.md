# Repo to Capability Matrix

## Purpose
This document defines a matrix for mapping repositories to the capabilities they require, strengthen, or can help manufacture across the lippytm ecosystem.

The goal is to make capability development more actionable by connecting it directly to repo surfaces rather than keeping it abstract.

## Suggested Capability Categories
- DevOps and rollout
- repo architecture and documentation
- integration and synchronization
- CRM and growth systems
- reporting and accountability
- education and learning systems
- media and narrative systems
- reasoning and diagnostics
- transparency and documentation governance

## Matrix Fields
For each relationship track:
- repo ID
- capability ID
- relationship type
- current maturity
- priority
- notes

## Example Relationship Types
- requires
- strengthens
- demonstrates
- teaches
- depends_on
- reusable_for

## Best Practices
- map high-priority repos first
- identify capabilities that improve multiple repos at once
- use this matrix to drive R&D, training, and promotion priorities
- revisit capability relationships after each wave or major architecture revision
