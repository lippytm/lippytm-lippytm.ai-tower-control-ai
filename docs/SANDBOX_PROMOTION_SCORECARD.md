# Sandbox Promotion Scorecard

## Purpose
This scorecard helps compare sandbox outputs and decide which experimental assets are strong enough for wider packaging or promotion.

## Score Areas
| Score Area | Description | Suggested Scale |
|---|---|---|
| Scenario Clarity | How clearly the sandbox purpose and scenario are defined | 1-5 |
| Repeatability | How reliably the sandbox can be rerun for useful learning | 1-5 |
| Diagnostic Value | How much useful insight the sandbox produces | 1-5 |
| Reuse Potential | How well the sandbox output could support multiple repos or domains | 1-5 |
| Packaging Readiness | How close the output is to becoming a template, module, or application component | 1-5 |
| Documentation and Transparency | How clearly notes, assumptions, and limits are handled | 1-5 |
| Promotion Value | How valuable wider promotion would be for the portfolio | 1-5 |

## Interpretation Guidance
- **29-35**: strong sandbox-promotion candidate
- **22-28**: good candidate with targeted improvement needed
- **15-21**: partial sandbox result, more shaping needed
- **below 15**: early-stage experimental path

## Best Practices
- score sandboxes before promotion into wider assets
- compare repeated runs to see if scores improve over time
- use this scorecard together with diagnostics packets and scenario packets
