# Repo to Revenue Opportunity Mapping Guide

## Purpose
This guide defines how to map repositories to concrete value and growth opportunities across the lippytm ecosystem.

The goal is to help show how each repo can support one or more value pathways without leaving those opportunities vague or disconnected from the rest of the factory.

## Core Mapping Questions
For each repository ask:
- what value does this repo create directly?
- what products, services, or applications can it support?
- what educational or media outputs can it strengthen?
- what CRM or reporting workflows can it improve?
- what platforms can deliver its outputs?
- what review level applies to those delivery surfaces?
- what related repos can reuse the same value path?

## Suggested Mapping Fields
- repo ID
- repo category
- linked product IDs
- linked service IDs
- linked application IDs
- linked platform IDs
- linked concept IDs
- linked component IDs
- value category
- maturity stage
- priority wave
- notes

## Suggested Value Categories
- product growth
- service growth
- application growth
- educational growth
- media growth
- CRM growth
- reporting growth
- cross-domain growth

## Best Practices
- map high-value repos first
- let one repo support multiple value categories where appropriate
- connect every mapped opportunity back to stable IDs and trackers
- distinguish internal support value from public delivery value
- review repo-to-value mappings after major architecture changes or new product ideas
