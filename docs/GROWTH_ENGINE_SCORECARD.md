# Growth Engine Scorecard

## Purpose
This scorecard helps compare repeatable growth engines across the lippytm ecosystem.

## Score Areas
| Score Area | Description | Suggested Scale |
|---|---|---|
| Asset Strength | Quality and usefulness of the underlying assets | 1-5 |
| Engine Clarity | How clearly the compounding growth path is defined | 1-5 |
| Cross-Domain Reuse | How well the engine supports more than one domain | 1-5 |
| Platform Flexibility | How well the engine adapts to different delivery surfaces | 1-5 |
| Documentation and Governance | How clearly review, documentation, and transparency are handled | 1-5 |
| Portfolio Leverage | How much the engine strengthens the broader portfolio | 1-5 |
| Expansion Potential | How easily the engine can be improved or widened later | 1-5 |

## Interpretation Guidance
- **29-35**: strong growth engine candidate
- **22-28**: good candidate with targeted improvement needed
- **15-21**: partial engine, more shaping needed
- **below 15**: early-stage path that should remain in development

## Best Practices
- score both single-engine and bundled-engine paths
- use the scorecard to decide which engines get earlier wave priority
- compare scores across product, service, application, AI, education, media, and CRM engines
