# Scientific Application Matrix

## Purpose
This document maps science-domain assets to portfolio application surfaces across the lippytm ecosystem.

## Application Surfaces
- repository assets
- products
- services
- applications
- learning modules
- media and narrative assets
- CRM and growth systems
- reporting and accountability systems
- AI systems and solutions

## Matrix Fields
For each scientific opportunity track:
- science domain ID
- opportunity ID
- target application surfaces
- reuse strength
- platform fit
- review level
- notes

## Best Practices
- map strongest science-to-application paths first
- distinguish reusable scientific core from surface-specific packaging
- connect scientific opportunities back to stable IDs and trackers
- use this matrix to prioritize later-wave scientific expansion
