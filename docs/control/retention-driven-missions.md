# Retention-Driven Missions

This document explains how the Control Tower should create missions from retention and repeat-engagement signals.

## Mission Triggers
- drop in repeat engagement
- active services with weak follow-up
- nurture backlog growth
- reengagement opportunities appearing repeatedly

## Mission Types
- follow-up quality mission
- retention routing mission
- reengagement mission
- expansion timing mission

## Best Practices
- use retention signals to improve timing and routing
- keep missions bounded and measurable
- prioritize higher-value active-service paths first
- turn repeated retention gaps into workflow and template improvements
