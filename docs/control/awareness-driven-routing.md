# Awareness-Driven Routing

This document explains how repo awareness, lane metadata, and promotion stage should influence routing decisions in the Control Tower.

## Goal

Route tasks and missions more accurately by using awareness data instead of relying only on keyword matches.

## Routing Factors
- primary lane
- secondary lane
- promotion stage
- value role
- risk level
- public surface status

## Routing Benefits
- better lane fit
- fewer cross-lane mistakes
- stronger readiness targeting
- better governance for higher-risk repos

## Best Practices
- use awareness before escalation where possible
- prefer the owning lane for domain-specific work
- route public-surface changes with extra care
- use promotion stage to limit premature automation
