# Evolutionary Advancement Scorecard

## Purpose
This scorecard helps compare evolutionary improvements and advancement paths across the lippytm ecosystem.

## Score Areas
| Score Area | Description | Suggested Scale |
|---|---|---|
| Advancement Clarity | How clearly the improvement path is defined | 1-5 |
| Strategic Value | How much the advancement strengthens the portfolio | 1-5 |
| Cross-Domain Impact | How well the advancement improves more than one area | 1-5 |
| Platform Flexibility | How well it adapts to multiple delivery surfaces | 1-5 |
| Documentation and Governance | How clearly tracking, review, and transparency are handled | 1-5 |
| Promotion Readiness | How ready the advancement is for broader rollout | 1-5 |
| Compounding Potential | How much future reuse or leverage it can create | 1-5 |

## Interpretation Guidance
- **29-35**: strong advancement candidate
- **22-28**: good advancement path with targeted improvement needed
- **15-21**: partial advancement, more shaping needed
- **below 15**: early-stage path that should remain in development

## Best Practices
- score both single-asset and cross-domain advancements
- use the scorecard to decide which improvements get earlier waves
- compare scorecards across repo, product, service, application, AI, education, media, and bundle paths
