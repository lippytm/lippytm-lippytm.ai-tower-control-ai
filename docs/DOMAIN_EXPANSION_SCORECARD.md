# Domain Expansion Scorecard

## Purpose
This scorecard helps evaluate how well a manufactured asset expands across multiple operating domains in the lippytm ecosystem.

## Score Areas
| Score Area | Description | Suggested Scale |
|---|---|---|
| Domain Coverage | Number and quality of domains supported | 1-5 |
| Reuse Strength | How reusable the asset is across domains | 1-5 |
| Platform Flexibility | How well it adapts to multiple platforms | 1-5 |
| Review Clarity | How clearly review levels stay aligned across uses | 1-5 |
| Documentation Quality | How well the asset is documented for reuse | 1-5 |
| Portfolio Value | How much portfolio-wide value the expansion creates | 1-5 |

## Interpretation Guidance
- **24-30**: strong multi-domain candidate
- **18-23**: good candidate with targeted improvement needed
- **12-17**: partial cross-domain fit
- **below 12**: narrow-scope asset for now

## Best Practices
- score assets before and after major reuse expansion
- use the scorecard to decide which assets deserve promotion into higher waves
- compare scorecards across product, service, application, and concept types
