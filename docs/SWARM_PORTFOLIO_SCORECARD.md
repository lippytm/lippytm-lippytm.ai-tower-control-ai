# Swarm Portfolio Scorecard

## Purpose
This scorecard provides a portfolio-level view of how well the lippytm swarm factory is performing across repositories, platforms, products, applications, and rollout waves.

## Score Areas
| Score Area | Description | Suggested Scale |
|---|---|---|
| Portfolio Coverage | How much of the repo/platform/application portfolio is mapped and tracked | 1-5 |
| Blueprint Reuse | How effectively blueprints are reused across scopes | 1-5 |
| Profile Coverage | How many important scopes have usable profiles | 1-5 |
| Product Mapping | How clearly repos and products are linked | 1-5 |
| Platform Fit | How well platform assignments match functions | 1-5 |
| Review Discipline | How clearly review levels and guardrails are applied | 1-5 |
| Improvement Throughput | How well backlog and improvement loops produce upgrades | 1-5 |
| Maturity Progress | How many priority items are moving toward repeatable or factory-grade | 1-5 |
| Cross-System Reuse | How well outputs are reused across education, business, media, CRM, and reporting | 1-5 |

## Interpretation Guidance
- **40-45**: strong portfolio operating foundation
- **30-39**: solid progress, more coverage and reuse needed
- **20-29**: partial portfolio system, prioritize first-wave completion
- **below 20**: portfolio architecture exists but deployment coverage is still limited

## Best Practices
- score after each wave or major architecture revision
- use the scorecard to prioritize portfolio-level improvements
- compare results across time, not only one snapshot
