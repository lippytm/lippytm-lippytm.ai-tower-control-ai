# Orchestration Scorecard

## Purpose
This scorecard helps evaluate how well ideas, projects, repositories, products, services, applications, and platforms are being orchestrated across the lippytm ecosystem.

## Score Areas
| Score Area | Description | Suggested Scale |
|---|---|---|
| Idea Clarity | How clearly ideas are defined and linked to projects | 1-5 |
| Project Structuring | How well projects are shaped into real manufacturing candidates | 1-5 |
| Repo Alignment | How clearly projects map to repo assets and manufactured inventory | 1-5 |
| Platform Fit | How well delivery platforms match the project | 1-5 |
| Portfolio Reuse | How much the orchestration strengthens reuse across the ecosystem | 1-5 |
| Documentation and Tracking | How well catalogs, trackers, and IDs support the work | 1-5 |
| Governance and Review | How clearly review levels and maturity are handled | 1-5 |

## Interpretation Guidance
- **29-35**: strong orchestration candidate
- **22-28**: good candidate with targeted improvement needed
- **15-21**: partial orchestration, more shaping needed
- **below 15**: early-stage path that should remain in development

## Best Practices
- score major ideas and projects before and after promotion
- use the scorecard to decide which pathways deserve earlier waves
- compare scorecards across product, service, application, and bundle paths
