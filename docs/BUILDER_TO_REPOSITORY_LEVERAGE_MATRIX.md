# Builder to Repository Leverage Matrix

## Purpose
This document maps builder systems to repository leverage opportunities across the lippytm ecosystem.

## Core Goal
Show how one strong builder can strengthen one or many repositories by improving their products, services, applications, AI solutions, sandboxes, training assets, or platform packages.

## Matrix Fields
For each leverage relationship track:
- builder ID
- source builder type
- target repo IDs
- leverage type
- reuse strength
- maturity impact
- notes

## Example Leverage Types
- accelerates documentation
- improves sandbox quality
- improves application packaging
- improves cross-domain reuse
- strengthens platform delivery
- strengthens training assets
- strengthens diagnostics and reasoning

- map highest-value repositories first
- use this matrix to identify which builders deserve earlier promotion
- compare leverage strength together with builder scorecards
- update leverage notes after major repo or platform changes
