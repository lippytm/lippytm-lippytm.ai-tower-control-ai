# R&D Innovation Governance Model

## Purpose
This document defines governance patterns for research, experimentation, capability growth, prototype promotion, and innovation scaling across the lippytm ecosystem.

The goal is to let innovation move quickly while still preserving clarity around review levels, promotion readiness, portfolio fit, and documentation quality.

## Core Governance Questions
For each R&D or innovation object ask:
- what is being explored?
- what systems could it improve?
- what review level applies?
- what evidence supports promotion?
- what risks or limits should remain visible?
- what portfolio wave should receive it?

## Governance Layers
### Layer 1 — Discovery Governance
Clarifies scope, themes, and research intent.

### Layer 2 — Prototype Governance
Keeps prototypes identifiable as prototypes and not final products prematurely.

### Layer 3 — Promotion Governance
Defines when an experiment or innovation is ready to become a product, service, application, or factory asset.

### Layer 4 — Portfolio Governance
Determines where innovations belong in the broader portfolio and which waves should carry them.

### Layer 5 — Improvement Governance
Ensures lessons learned return to templates, scorecards, trackers, and capability systems.

## Best Practices
- keep exploratory objects separate from production-grade objects
- make promotion evidence explicit
- document review level and maturity together
- link innovation governance back to scorecards and trackers
- prefer repeatable improvements over isolated novelty
