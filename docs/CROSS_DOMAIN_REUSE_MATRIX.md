# Cross-Domain Reuse Matrix

## Purpose
This document defines a matrix for tracking how one repo, product, service, component, concept, or application can be reused across multiple operating domains in the lippytm ecosystem.

## Domains
- software and repo operations
- products and services
- education and learning
- CRM and growth
- reporting and accountability
- media and narrative
- synchronization and interoperability
- reasoning and documentation

## Matrix Fields
For each reusable object track:
- object ID
- object type
- source domain
- target domains
- reuse strength
- review level
- notes

## Best Practices
- track the strongest reuse paths first
- make target-domain reuse explicit in catalogs and profiles
- use this matrix to prioritize cross-domain manufacturing candidates
- revisit reuse mappings after each major wave
