# R&D Application Builder Scorecard

## Purpose
This scorecard helps compare R&D application builders across the lippytm ecosystem so stronger builder systems can be identified, improved, and promoted earlier.

## Score Areas
| Score Area | Description | Suggested Scale |
|---|---|---|
| Builder Clarity | How clearly the builder purpose, modules, and boundaries are defined | 1-5 |
| Research Linkage | How well the builder connects to research inputs, sandbox assets, and diagnostics | 1-5 |
| Reuse Potential | How well the builder can support multiple repos, projects, or domains | 1-5 |
| Platform Fit | How well the builder can be adapted to one or more delivery surfaces | 1-5 |
| Packaging Readiness | How close the builder is to becoming a packaged application or reusable system | 1-5 |
| Documentation and Governance | How clearly assumptions, review levels, transparency, and tracking are handled | 1-5 |
| Portfolio Leverage | How much the builder strengthens the broader portfolio | 1-5 |

## Interpretation Guidance
- **29-35**: strong builder candidate for broader rollout
- **22-28**: good builder with targeted improvement needed
- **15-21**: partial builder, more shaping needed
- **below 15**: early-stage builder path

## Best Practices
- score builders before and after major promotion steps
- compare scores across reasoning, diagnostics, workflow, multimedia, database, learning, and cross-domain builders
- use this scorecard together with repeatability and sandbox-promotion signals
