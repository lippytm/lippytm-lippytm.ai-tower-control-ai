# Platform Capability Maturity Matrix

## Purpose
This document combines platform-fit guidance with maturity stages so the lippytm ecosystem can understand not only what each platform is good at, but how mature a given capability is on that platform.

- MyClaw
- ChatGPT / OpenAI
- Claude

- `concept`
- `blueprinted`
- `profiled`
- `operational`
- `repeatable`
- `factory_grade`

## How to Use This Matrix
- identify strong-fit platforms for each capability
- identify which capabilities are ready for broader rollout
- identify weak or early-stage platform pairings
- prioritize repeatable or factory-grade combinations first

## Best Practices
- improve maturity status only when repeatability is demonstrated
- treat platform maturity as dynamic and revisable
- use this matrix to guide deployment-wave sequencing
- keep GitHub-linked documentation current as the authoritative reference
