# Documentation Taxonomy and Ownership Register

## Purpose
This document defines a taxonomy and ownership model for documentation assets across the lippytm ecosystem so docs can be organized, maintained, and improved more reliably over time.

## Documentation Taxonomy
### Architecture Docs
System models, strategy docs, and operating patterns.

### Templates
Reusable forms for profiles, packets, scorecards, and registers.

### Registers and Catalogs
Structured records for inventory, products, concepts, sync objects, and applications.

### Trackers and Status Files
Coverage, rollout, maturity, review, and improvement visibility files.

### Packets and Reports
Reasoning packets, review packets, story packets, reporting packets, and media packets.

### Narrative and Media Docs
Storyline docs, storyboard packets, continuity registers, and media planning docs.

## Best Practices
- assign an owner for important docs even if updates are collaborative
- separate doc type from doc status
- review high-value docs on a regular cadence
- link doc ownership to trackers and improvement notes where appropriate
