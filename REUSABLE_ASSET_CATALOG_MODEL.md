# Reusable Asset Catalog Model

This document defines how the wider `lippytm` ecosystem can keep reusable assets organized as the system grows across study tracks, toolkit packs, sandbox systems, guides, kits, products, memberships, and affiliate paths.

## Purpose
Make reusable assets easier to find, compare, review, update, and promote by giving the ecosystem a clearer cataloging model.

## Core Principle
A growing ecosystem becomes easier to maintain when reusable assets are visible as distinct units with clear roles, maturity, review status, and reuse paths.

## Asset Types

### 1. Learning Assets
Use for:
- lessons
- study tracks
- teaching guides
- review guides

### 2. Practice Assets
Use for:
- sandbox paths
- practice loops
- small exercises
- repeatable builder tasks

### 3. Support Assets
Use for:
- setup guides
- toolkit notes
- dependency notes
- troubleshooting notes

### 4. Product Assets
Use for:
- packs
- kits
- bundles
- member-facing materials
- affiliate-ready materials

## Core Catalog Fields
Each strong reusable asset should make it easy to see:
- purpose
- audience
- setup path
- review path
- reuse path
- maturity level
- support status

## Best Practices
- catalog only assets that are clear enough to reuse
- keep asset descriptions short and comparable
- connect asset status back to review and health-check systems
- archive weak or outdated assets clearly
- promote durable repeated assets into stronger packs, templates, or standards
