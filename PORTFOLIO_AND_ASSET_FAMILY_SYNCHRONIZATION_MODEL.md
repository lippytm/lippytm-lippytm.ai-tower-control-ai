# Portfolio and Asset Family Synchronization Model

This document defines how the wider `lippytm` ecosystem can keep aligned portfolios and grouped asset families synchronized as the system expands.

## Purpose
Help the ecosystem avoid drift between high-level portfolio structures and the concrete reusable asset families that support learning, practice, workflows, support, and products.

## Core Principle
A portfolio should stay useful only if the asset families beneath it remain clear, current, and connected to the same review, rollout, and support expectations.

## Synchronization Areas

### 1. Learning Synchronization
Keep aligned:
- learning portfolios
- lesson families
- study tracks
- curriculum guides

### 2. Practice Synchronization
Keep aligned:
- practice portfolios
- practice loops
- sandbox families
- repeatable builder tasks

### 3. Support Synchronization
Keep aligned:
- support portfolios
- setup guides
- troubleshooting families
- environment notes

### 4. Workflow Synchronization
Keep aligned:
- workflow portfolios
- workflow guides
- agent pattern families
- secure workflow notes

### 5. Product Synchronization
Keep aligned:
- product portfolios
- packs
- kits
- bundles
- member-facing and affiliate-ready assets

## Synchronization Signals
A portfolio and asset family are well synchronized when:
- the linked assets are still current
- support and review expectations match
- readiness and rollout status are visible
- overlapping weak members can be simplified or archived
- the structure improves navigation instead of adding clutter

## Best Practices
- review synchronization in small loops
- connect synchronization checks back to health checks and readiness models
- update portfolio links when asset families change materially
- preserve useful specialization while reducing drift and duplication
- treat synchronization as a way to strengthen clarity, not add more bureaucracy
