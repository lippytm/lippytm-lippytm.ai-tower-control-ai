# Trust and Compliance

## Core Principles
- no guaranteed income claims
- no deceptive marketing
- no fabricated testimonials
- no hidden refund assumptions
- clear deliverables and role boundaries
- protect user and client data
- require human review for sensitive decisions

## Ecosystem Rules
- public hubs explain clearly
- ops hubs document workflows clearly
- education hubs separate learning content from legal, tax, or financial advice
- private automation hubs protect sensitive operational details

## Documentation Standards
Every priority repo should eventually include:
- README
- PROJECT_ROLE or equivalent
- SYSTEM_ARCHITECTURE or equivalent
- RUNBOOK
- TRUST_AND_COMPLIANCE or equivalent
- links back to the control tower

## Sensitive Areas
- funding and financial topics
- legal-adjacent content
- private data handling
- automated outreach and messaging
- security-sensitive automations

## Goal
Create a flexible but trustworthy ecosystem where creativity, education, automation, and business development can grow without losing clarity or ethical standards.
