# Maturity and Promotion Pathways

This document defines how the wider `lippytm` ecosystem can recognize when tracks, packs, guides, kits, and variants are ready to move into a stronger reusable role.

## Purpose
Give the ecosystem a repeatable way to decide when something should stay local, become reusable, be promoted into a standard, or be retired.

## Core Principle
Not every new pattern should be promoted. Promotion should happen when a pattern becomes clear, durable, reusable, and helpful across more than one local case.

## Maturity Levels

### 1. Local Level
Use for:
- local experiments
- local notes
- early versions
- narrow fixes

### 2. Repeatable Level
Use for:
- repeated lessons
- repeated setup paths
- repeated workflow steps
- repeated review patterns

### 3. Reusable Level
Use for:
- template-ready patterns
- pack-ready patterns
- guide-ready patterns
- reusable study-track structures

### 4. Standard Level
Use for:
- control-tower guidance
- shared defaults
- stronger ecosystem-wide review expectations
- clearer packaging and maintenance rules

## Promotion Signals
Promote when:
- the same pattern works in multiple contexts
- setup and review notes are clear
- security and maintenance expectations are visible
- the pattern reduces confusion instead of adding clutter
- weaker alternatives can be replaced or archived

## Best Practices
- keep maturity criteria simple enough to repeat
- promote only patterns that prove durable
- link promoted patterns back to health checks and review loops
- preserve useful local flexibility while reducing repeated confusion
- archive weaker alternatives when the promoted path is clearly better
