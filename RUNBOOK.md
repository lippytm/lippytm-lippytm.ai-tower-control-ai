# Runbook

## Daily
- review open priorities
- review repo health and new changes
- review documentation gaps
- update the next actions for primary hubs

## Weekly
- review `PROJECT_INDEX.md`
- review `docs/architecture/FLEET_STATUS.md`
- review architecture drift across priority repos
- review missing documentation in anchor repos
- review educational and business development priorities

## Monthly
- audit repo roles
- archive stale concepts into reference repos where needed
- refresh public ecosystem descriptions
- refresh internal ops documentation
- review flexibility, diversity, and overlap across the fleet

## Incident Handling
1. identify affected repo or workflow
2. pause risky automation if needed
3. document the problem
4. update the relevant architecture or runbook doc
5. link the fix back to the control tower if it affects multiple repos
