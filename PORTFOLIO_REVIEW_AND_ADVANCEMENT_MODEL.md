# Portfolio Review and Advancement Model

This document defines how the wider `lippytm` ecosystem can review and advance aligned portfolios over time.

## Purpose
Help the ecosystem keep portfolio structures useful by giving them a clearer path for review, simplification, promotion, support, and advancement.

## Core Principle
A portfolio should not be treated as a static grouping. It should be reviewed as a living structure that can improve, simplify, expand, or retire weaker parts.

## Portfolio Review Areas

### 1. Learning Portfolio Review
Review:
- lesson clarity
- study-track usefulness
- curriculum alignment
- repeated confusion that suggests stronger shared guidance

### 2. Practice Portfolio Review
Review:
- durability of practice loops
- usefulness of sandbox paths
- repeatability of builder exercises
- whether practice structures should be promoted into stronger packs

### 3. Support Portfolio Review
Review:
- clarity of setup guides
- stability of troubleshooting notes
- overlap between support assets
- whether support guidance should become stronger standards

### 4. Workflow Portfolio Review
Review:
- maturity of workflow guides
- secure workflow boundaries
- overlap between workflow notes and runbook-like structures
- whether repeated workflow assets should be simplified or promoted

### 5. Product Portfolio Review
Review:
- pack quality
- supportability of kits and bundles
- readiness of member-facing and affiliate-ready assets
- whether weaker overlapping products should be archived or simplified

## Advancement Signals
Advance a portfolio when:
- grouped assets reduce confusion instead of adding it
- review and support expectations are clearer together
- repeated assets can be simplified into stronger reusable forms
- broader rollout becomes easier because maturity is visible
- weaker members can be archived without losing important capability

## Best Practices
- review portfolios in small loops instead of rare large audits
- connect portfolio advancement back to health checks, readiness models, and promotion models
- simplify weak overlap before expanding the portfolio further
- preserve useful specialization while reducing avoidable clutter
- use advancement to strengthen reuse, not just enlarge the portfolio count
