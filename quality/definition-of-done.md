# Definition of Done

Quality is Job #1.

A task, workflow, file, automation, prompt, issue, product package, or repo upgrade is not complete until it is clear, reviewable, safe, and useful.

## Universal Definition of Done

- [ ] The purpose is clear.
- [ ] The audience or user is clear.
- [ ] The expected input is clear.
- [ ] The expected output is clear.
- [ ] The next action is clear.
- [ ] The risk level is identified.
- [ ] Security/privacy concerns are reviewed.
- [ ] No secrets are committed.
- [ ] No unsupported legal, tax, funding, investment, trading, medical, or income guarantees are included.
- [ ] Human review is required for high-risk actions.
- [ ] A support path or fallback exists.
- [ ] A test, validation step, or manual review step exists.
- [ ] The work is connected to a repo, workflow, product, service, support process, or dashboard.

## Documentation Done

- [ ] File has a clear title.
- [ ] File explains its purpose.
- [ ] File includes practical next steps.
- [ ] File avoids vague hype without implementation value.
- [ ] File can be reused by ChatGPT, GitHub Copilot, Twin, or future agents.

## Automation Done

- [ ] Trigger is defined.
- [ ] Input is defined.
- [ ] Output is defined.
- [ ] Owner or review point is defined.
- [ ] Failure mode is defined.
- [ ] Manual fallback is defined.
- [ ] Sensitive actions require human approval.

## AI Coding Done

- [ ] User story is clear.
- [ ] Files to create or edit are listed.
- [ ] Smallest useful implementation is identified.
- [ ] Tests or validation steps are included.
- [ ] Docs are updated.
- [ ] Security and privacy are checked.
- [ ] Rollback or undo path is described when needed.

## Productization Done

- [ ] Customer type is identified.
- [ ] Problem solved is clear.
- [ ] Deliverables are clear.
- [ ] Offer language is realistic.
- [ ] Support expectations are clear.
- [ ] Monetization path is identified.
- [ ] Affiliate links are labeled correctly when used.

## Dashboard Done

- [ ] Status is clear.
- [ ] Next action is clear.
- [ ] Risk or blocker is clear.
- [ ] Owner or responsible role is clear.
- [ ] Last updated date or review cadence is included when useful.
