# Navigation Stack Alignment Model

This document defines how the wider `lippytm` ecosystem can align portfolios, asset families, study tracks, packs, guides, and products into a clearer navigation stack.

## Purpose
Help the ecosystem stay easier to explore by making high-level structures and concrete reusable assets work together as one navigation system instead of many disconnected layers.

## Core Principle
A large modular ecosystem becomes easier to use when people can move clearly from high-level portfolios to asset families, from asset families to tracks or packs, and from tracks or packs to specific reusable materials.

## Navigation Layers

### 1. Portfolio Layer
Use for:
- broad learning areas
- broad practice areas
- broad support areas
- broad workflow areas
- broad product areas

### 2. Asset Family Layer
Use for:
- grouped lessons
- grouped practice loops
- grouped support notes
- grouped workflow assets
- grouped product assets

### 3. Track and Pack Layer
Use for:
- study tracks
- specialization tracks
- reusable packs
- reusable kits
- bundle-ready structures

### 4. Asset Layer
Use for:
- specific lessons
- specific guides
- specific sandbox paths
- specific setup notes
- specific reusable materials

## Alignment Signals
The navigation stack is strong when:
- each layer points clearly to the next layer down
- users can move from broad structure to concrete asset without confusion
- support and review expectations stay visible at each layer
- overlapping weak paths can be simplified or archived
- the stack improves reuse and discovery instead of adding clutter

## Best Practices
- keep each layer simpler than the one beneath it
- use the stack to guide growth instead of duplicating the same structure in many places
- connect stack updates back to health checks, readiness, and promotion models
- preserve useful specialization while reducing avoidable confusion
- review the stack regularly as portfolios and asset families change
