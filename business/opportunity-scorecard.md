# Opportunity Scorecard

Purpose: score ideas, platforms, repos, workflows, affiliate links, and product offers before spending time or money building them.

Quality is Job #1. Build viable money paths before complex platforms.

## Score each idea from 1 to 10

| Factor | Score | Notes |
|---|---:|---|
| Speed to launch |  | Can this be tested quickly? |
| Revenue potential |  | Can this create sales, commissions, or support revenue? |
| Cost to build |  | Lower cost earns higher score. |
| Skill requirement |  | Lower barrier earns higher score. |
| Platform dependency |  | Less lock-in earns higher score. |
| Customer demand |  | Is this a real problem people understand? |
| Repeatability |  | Can this become a template/package? |
| Support burden |  | Lower support burden earns higher score. |
| Legal/financial risk |  | Lower risk earns higher score. |
| Automation potential |  | Can AI/tools repeat parts of it? |
| Brand fit |  | Does it fit lippytm.ai Business-of-Businesses? |
| Long-term asset value |  | Does it improve the fleet over time? |

## Tier rules

### Tier 1: Build Now

Simple, monetizable, lower risk, useful immediately.

Examples:
- AI Business Automation Starter Kit
- AI Coding Repo Upgrade Kit
- Affiliate Automation Engine
- AI Support Desk Starter
- ManyChat/BotBuilders intake flow

### Tier 2: Build After Proof

Useful but needs early validation.

Examples:
- full dashboards
- client portals
- multi-agent CRM
- automated reporting apps
- advanced integration hubs

### Tier 3: Research and Development

High imagination, higher complexity, future asset value.

Examples:
- robotics command hubs
- Web3 marketplace systems
- virtual property automation
- AI education worlds
- decentralized knowledge networks

## Money Path First questions

Before building, answer:

1. Who pays or benefits?
2. What problem is solved?
3. What result is delivered?
4. Can it be sold manually first?
5. Can it become a template or service package?
6. What is the support burden?
7. What is the risk level?
8. What is the simplest test?
