# Toolkit and Sandbox Review Model

This document defines how support toolkits and sandbox systems should be reviewed across the wider `lippytm` ecosystem.

## Purpose
Keep language packs, toolkit guides, machine learning practice systems, blockchain practice systems, workflow sandboxes, and reflection environments useful, safe, and maintainable over time.

## Core Principle
A toolkit or sandbox should be reviewed not only for technical usefulness, but also for clarity, safety, reuse value, and supportability.

## Review Areas

### 1. Setup Review
Review:
- setup clarity
- dependency notes
- environment assumptions
- reproducibility of the setup path

### 2. Learning Review
Review:
- lesson usefulness
- exercise clarity
- whether the sandbox teaches the intended concept
- whether the toolkit supports real builder tasks

### 3. Security Review
Review:
- secrets and credential handling notes
- sandbox vs production separation
- scope boundaries
- review checkpoints for risky actions

### 4. Reuse Review
Review:
- whether the toolkit or sandbox can become a reusable pack
- whether repeated steps can become templates
- whether setup and review notes are transferable

### 5. Maintenance Review
Review:
- update visibility
- stale dependency risk
- unclear instructions
- outdated examples

## Best Practices
- keep reviews small and repeatable
- review setup paths before multiplying more variants
- connect review findings back to templates and kits
- archive confusing or weak sandbox patterns instead of letting them drift
- promote durable toolkit paths into reusable specialization packs
