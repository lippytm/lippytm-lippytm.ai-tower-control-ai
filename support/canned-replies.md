# Canned Replies

Reusable support replies for lippytm.ai automation, chatbot, AI coding, affiliate, funding intake, and repo modernization workflows.

Quality is Job #1. Review before sending. Avoid unsupported guarantees.

---

## General acknowledgement

```text
Thanks for reaching out — I can help organize the next step.

Here is what I understand so far:
[summary]

The best next action is:
[next action]
```

## Need more information

```text
I can help with that. To route this correctly, I need the basic context:

1. What platform is involved?
2. What are you trying to do?
3. What happened instead?
4. Is this related to leads, automation, chatbot setup, funding intake, website, GitHub, or support?
```

## Funding intake safe reply

```text
I can help organize your business funding intake information and prepare the next steps.

Approvals, terms, rates, and outcomes depend on provider review, qualifications, documentation, and application details. I do not want to promise an outcome, but I can help organize the process and route the information properly.
```

## Technical issue reply

```text
Thanks — this sounds like a technical setup issue.

Please check:
- platform involved
- exact error message
- what was expected
- what happened instead
- whether credentials, API keys, or private data are involved

Do not paste API keys, passwords, or private credentials into the message.
```

## Repo upgrade reply

```text
A repo upgrade can help make the project clearer for humans and AI coding tools.

The usual upgrade includes README, roadmap, Copilot instructions, issue templates, pull request template, quality checklist, security notes, and productization direction.
```

## Escalation reply

```text
This may involve a higher-risk area such as billing, private data, funding, legal/tax/financial claims, credentials, or public-facing publication.

The safest next step is human review before taking action.
```
