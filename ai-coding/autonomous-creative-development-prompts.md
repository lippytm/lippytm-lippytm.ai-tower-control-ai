## 9. Automation Boundaries

### Safe automatic drafting

These may be drafted automatically after setup:

- issues
- docs
- prompts
- checklists
- repo summaries
- content drafts
- PR descriptions
- support scripts
- README improvement plans

### Human review required

These require review before execution or publication:

- code changes
- workflow/deployment changes
- customer-facing publications
- financial, funding, legal, tax, investment, trading, or income claims
- API integrations
- secret or credential handling
- payment or billing flows
- robotics or physical-world actions
