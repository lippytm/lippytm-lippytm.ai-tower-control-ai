# Contributing

This repo is the lippytm.ai Control Tower for AI coding, autocomplete automations, agents, workflows, dashboards, support, productization, and fleet modernization.

Quality is Job #1.

## Contribution workflow

1. Open or use a GitHub issue before major changes.
2. Identify the connected area: AI coding, automation, support, platform, integration, product, marketing, dashboard, or security.
3. Keep changes small and reviewable.
4. Update related registries or dashboards when relevant.
5. Follow the Definition of Done.
6. Use the Pull Request template.

## Preferred changes

Good contributions include:

- clearer docs
- safer prompts
- better workflows
- stronger support scripts
- registry updates
- dashboard updates
- issue templates
- quality/security improvements
- product/service package improvements
- autocomplete/Copilot guidance

## Avoid

Do not add:

- real secrets
- private customer data
- unsupported financial, funding, legal, tax, investment, trading, medical, or income guarantees
- hidden behavior
- unsafe deployment instructions
- unclear API credential handling

## Human review required

Human review is required for:

- code deployments
- customer-facing publications
- funding/finance/legal/tax/investment/trading claims
- payment or billing workflows
- API integrations
- customer data workflows
- robotics or physical-world actions

## Commit style

Use clear commit messages:

```text
Add support routing workflow
Update platform registry
Add Twin agent definition
Improve funding intake safety language
```

## Definition of done

Before marking work complete:

- [ ] Purpose is clear
- [ ] Risk level is clear
- [ ] No secrets are included
- [ ] No unsupported claims are included
- [ ] Support/fallback path is considered
- [ ] Related dashboard/registry updated if relevant
