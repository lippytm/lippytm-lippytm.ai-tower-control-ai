# Portfolio to Track and Asset Linking Model

This document defines how aligned portfolios in the wider `lippytm` ecosystem should connect back to specific asset families, study tracks, packs, guides, and reusable systems.

## Purpose
Help the ecosystem keep portfolio structures practical by making it clearer how large portfolio groupings relate to concrete reusable assets and future track-building work.

## Core Principle
A portfolio is most useful when it helps people find the specific tracks, assets, packs, and guides that do the real work instead of remaining only a high-level concept.

## Linking Areas

### 1. Learning Portfolio Linking
Connect to:
- lessons
- study tracks
- curriculum maps
- teaching guides

### 2. Practice Portfolio Linking
Connect to:
- practice loops
- sandbox paths
- repeatable builder tasks
- review routines

### 3. Support Portfolio Linking
Connect to:
- setup guides
- toolkit notes
- troubleshooting notes
- environment notes

### 4. Workflow Portfolio Linking
Connect to:
- workflow guides
- agent patterns
- secure workflow notes
- runbook-like structures

### 5. Product Portfolio Linking
Connect to:
- packs
- kits
- bundles
- member-facing assets
- affiliate-ready assets

## Linking Signals
A portfolio link is strong when:
- the linked assets are easy to identify
- the linked tracks reduce confusion instead of adding it
- support and review expectations remain visible
- the portfolio improves navigation into reusable assets
- weak overlapping links can be simplified or archived

## Best Practices
- keep portfolio links concrete and short
- connect portfolios back to asset-family, readiness, and promotion models
- use portfolio links to guide future track-building instead of duplicating track definitions
- preserve useful specialization while reducing avoidable overlap
- review portfolio links as part of normal ecosystem health checks
