# Cybersecurity Education Track

This document defines cybersecurity as a core applied learning track within the wider `lippytm` AI education ecosystem.

## Purpose
Treat cybersecurity as an always-applied discipline across AI development, software engineering, automation, mixed environments, and educational systems.

## Core Objectives
- teach safe computing habits early
- teach secure development habits alongside coding
- teach risk awareness for automation and agent systems
- teach privacy, secrets handling, and access control basics
- teach documentation, review, and incident-response habits
- make cybersecurity part of everyday builder practice

## Core Domains

### 1. Secure Development Habits
Topics:
- least privilege thinking
- secrets management
- dependency awareness
- safe defaults
- code review habits
- documentation of assumptions and risks

### 2. Operational Security Habits
Topics:
- access control
- credential safety
- workflow permissions
- audit trails
- environment separation
- safer automation boundaries

### 3. AI and Agent Security
Topics:
- prompt and workflow boundaries
- human review points
- data-handling caution
- failure-case documentation
- sandbox vs production separation

### 4. Builder Self-Protection
Topics:
- phishing awareness
- suspicious dependency awareness
- environment hygiene
- secure backup habits
- review before execution

### 5. Educational Integration
Topics:
- include security notes in lessons
- include safer defaults in templates
- include failure cases as teaching material
- include review checklists in reusable kits

## Best Practices
- teach security with every build track
- document risks, not just features
- keep production and experiments separated
- use small examples that model safer behavior
- make security review part of learning loops
