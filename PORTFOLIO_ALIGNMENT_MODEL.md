# Portfolio Alignment Model

This document defines how the wider `lippytm` ecosystem can align reusable assets, study tracks, packs, kits, and products into clearer portfolios as the system grows.

## Purpose
Help the ecosystem group mature reusable work into clearer portfolios so growth stays easier to navigate, review, support, and improve.

## Core Principle
A large modular ecosystem becomes easier to manage when related reusable assets are aligned into portfolios with clear roles, audiences, maturity levels, and reuse paths.

## Portfolio Types

### 1. Learning Portfolio
Use for:
- lessons
- study tracks
- curriculum maps
- teaching guides

### 2. Practice Portfolio
Use for:
- practice loops
- sandbox paths
- repeatable exercises
- builder review routines

### 3. Support Portfolio
Use for:
- setup guides
- toolkit notes
- troubleshooting notes
- environment notes

### 4. Workflow Portfolio
Use for:
- agent patterns
- workflow guides
- secure workflow notes
- runbook-like structures

### 5. Product Portfolio
Use for:
- packs
- kits
- bundles
- member-facing assets
- affiliate-ready assets

## Alignment Signals
Align assets into stronger portfolios when:
- they share the same audience or role
- they reduce repeated confusion when grouped
- support and review expectations become clearer together
- overlapping weaker versions can be simplified or archived
- the grouped portfolio improves reuse instead of adding clutter

## Best Practices
- align only when the portfolio reduces confusion
- keep portfolio descriptions short and clear
- connect portfolio alignment back to catalog, readiness, and promotion guidance
- preserve useful specialization inside the portfolio structure
- archive weak overlapping portfolio members clearly
