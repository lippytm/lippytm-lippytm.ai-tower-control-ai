# Asset Family Rollout and Support Guide

This document defines how grouped asset families in the wider `lippytm` ecosystem should connect to rollout, support, refresh, and promotion decisions.

## Purpose
Help the ecosystem manage reusable assets more clearly once they are grouped into learning, practice, support, workflow, and product families.

## Core Principle
Grouping assets is only useful when the family structure also makes rollout, support, and improvement decisions easier.

## Family Decision Areas

### 1. Learning Family Decisions
Review:
- which lessons are experimental
- which study tracks are ready for broader reuse
- which curriculum assets should be promoted into stronger reusable guides

### 2. Practice Family Decisions
Review:
- which practice loops are durable
- which sandbox paths should remain experimental
- which repeated exercises should be promoted into packs or templates

### 3. Support Family Decisions
Review:
- which setup guides are stable enough for wider use
- which troubleshooting notes should become stronger standards
- which environment notes should remain local or specialized

### 4. Workflow Family Decisions
Review:
- which workflow guides are mature enough for broader rollout
- which agent patterns should remain sandboxed
- which runbook-like structures should become stronger reusable guidance

### 5. Product Family Decisions
Review:
- which packs are ready for productization
- which bundles are supportable enough for broader rollout
- which member or affiliate assets should remain limited until clearer support exists

## Best Practices
- connect family review back to readiness and maturity models
- keep rollout decisions visible at the asset-family level
- promote durable repeated family patterns into clearer standards and packs
- archive weak overlapping family members clearly
- use grouping to reduce confusion, not add more bureaucracy
